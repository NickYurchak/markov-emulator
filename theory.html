Основні теоретичні відомості по НАМ

Нормальні алгоритми Маркова (НАМ) є формальною системою перетворення рядків, запропонованою радянським математиком Андрієм Марковим у 1940-х роках. НАМ використовують набір правил підстановок для обробки рядків, де кожне правило складається з зразка та його заміни. Якщо зразок знаходиться у рядку, він замінюється на відповідну заміну. Процес продовжується доки не залишиться правил підстановок, які можна застосувати, або не буде застосовано термінальне правило.

Термінальні правила вводяться додаванням символу крапки в кінці заміни. Якщо заміна містить крапку в кінці, це означає, що алгоритм зупиняється після застосування цього правила.

НАМ складається з:

  • Алфавіт: Набір символів, що використовуються у рядках.
  • Правила підстановок: Набір пар "зразок-заміна", які визначають, як перетворювати рядки.
  • Робочий рядок: Початковий рядок, до якого застосовуються правила підстановок.

Інструкція з використання емулятора

  1. Введіть початковий рядок у поле "Робочий рядок".
  2. Введіть правила підстановок у таблицю. Для цього додайте рядки за допомогою кнопок "Додати рядок вище" ( ) або "Додати рядок нижче" ( ) і заповніть комірки "Зразок" та "Заміна". Комірки "Коментар" є необов'язковими та можуть використовуватися для пояснень.
  3. Для запуску алгоритму натисніть кнопку "Запустити алгоритм" (). Використовуйте кнопку "Виконати один крок" () для покрокового виконання або кнопку "Запустити алгоритм до завершення" () для повного виконання.
  4. При необхідності змініть швидкість виконання алгоритму за допомогою повзунка "Швидкість виконання". Швидкість виконання можна змінювати навіть протягом роботи алгоритму.
  5. Встановіть максимальну кількість ітерацій у випадаючому списку "Максимальна кількість ітерацій".
  6. Для очищення робочого рядка та таблиці правил натисніть кнопку "Очистити" ().
  7. Для експорту введених правил та робочого рядка у файл натисніть кнопку "Експортувати".
  8. Для імпорту правил та робочого рядка з файлу натисніть кнопку "Імпортувати".
  9. При натисканні кнопки "Побудувати графік складності" відбувається перевірка введених даних. Якщо дані коректні, програма обчислює кількість ітерацій для різних довжин слів і будує графік, що відображає складність алгоритму. Якщо введені дані некоректні, користувач отримує відповідні повідомлення про помилки.
  10. Автоматичне виконання алгоритму можна призупинити, натиснувши на ту ж саму кнопку, за допомогою якої він був запущений. Іконка на кнопці змінюється на іконку паузи, а після зупинки можна продовжити виконання алгоритму будь-яким з доступних методів.
  11. Програма підсвічує зеленим кольором правило підстановки, що виконується в даний момент, дозволяючи користувачам легко відстежувати процес виконання алгоритму.
  12. Для відображення процесу виконання алгоритму та його аналізу використовується логування. Логи можна переглядати у спеціальному контейнері, який відкривається натисканням кнопки "Логування".
